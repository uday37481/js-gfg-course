

1. Introduction to Cloud Computing

Cloud Computing is a technology that delivers computing services such as servers, storage, databases, networking, software, and analytics over the internet (“the cloud”). Instead of owning and maintaining physical hardware, users can access computing resources on demand from cloud service providers.

Cloud computing provides:

Scalability
Flexibility
Cost efficiency
High availability
Faster deployment

It allows businesses and individuals to use IT resources whenever needed and pay only for what they use.


2. Key Terminologies in Cloud Computing

2.1 Cloud

A cloud is a collection of servers, storage systems, and networking resources connected through the internet to provide computing services remotely.

Features
Accessible from anywhere
Shared resources
On-demand services
Managed by providers

2.2 Virtualization

Virtualization is the process of creating virtual versions of physical resources such as servers, storage devices, or networks.

Types of Virtualization
Server Virtualization
Storage Virtualization
Network Virtualization
Desktop Virtualization

Benefits
Better hardware utilization
Reduced cost
Easy management
Improved scalability

2.3 Hypervisor

A hypervisor is software that creates and manages virtual machines (VMs).

Types
Type 1 Hypervisor (Bare Metal)
Runs directly on hardware
Example: VMware ESXi, Hyper-V
Type 2 Hypervisor
Runs on an operating system
Example: VirtualBox

2.4 Virtual Machine (VM)

A Virtual Machine is a software-based computer that runs an operating system and applications like a physical computer.

Advantages
Isolation
Flexibility
Easy backup and recovery
Multiple OS support

2.5 Containerization

Containerization packages applications and their dependencies into lightweight containers.

Popular Container Platforms
Docker
Kubernetes

Benefits
Fast deployment
Portability
Efficient resource usage

2.6 Scalability

Scalability is the ability of a cloud system to increase or decrease resources according to workload demand.

Types
Vertical Scaling
Increase hardware capacity
Horizontal Scaling
Add more machines/servers

2.7 Elasticity

Elasticity is the automatic adjustment of cloud resources based on real-time demand.

Example

During heavy website traffic, additional servers are automatically added and removed later when traffic decreases.

2.8 Multi-Tenancy

Multi-tenancy means multiple users share the same cloud infrastructure securely.

Benefits
Cost reduction
Efficient resource usage
Easy maintenance

2.9 Pay-as-You-Go

Users pay only for the cloud resources they consume.

Advantages
No upfront investment
Budget-friendly
Cost optimization

2.10 SLA (Service Level Agreement)

An SLA is a contract between the cloud provider and customer defining:

Service availability
Performance
Responsibilities
Security guarantees

2.11 Data Center

A data center is a facility containing servers, storage systems, networking equipment, and cooling systems.

Cloud providers maintain large global data centers.

2.12 Load Balancing

Load balancing distributes incoming traffic across multiple servers.

Benefits
Improved performance
High availability
Fault tolerance

2.13 Fault Tolerance

Fault tolerance is the ability of a system to continue operating even when failures occur.

2.14 Disaster Recovery

Disaster Recovery (DR) refers to backup and recovery methods used to restore services after failures or disasters.

2.15 Cloud Deployment Models

1. Public Cloud
Services are available over the public internet.
Example:
AWS
Azure

2. Private Cloud
Cloud infrastructure is dedicated to a single organization.

3. Hybrid Cloud
Combination of public and private cloud.

4. Community Cloud
Shared among organizations with similar requirements.


3. Cloud Service Models

3.1 IaaS (Infrastructure as a Service)

Provides virtualized computing resources.

Services
Virtual machines
Storage
Networking

Examples
Amazon EC2
Azure Virtual Machines

Benefits
Full control
Flexible infrastructure

3.2 PaaS (Platform as a Service)

Provides a platform for application development and deployment.

Services
Runtime environment
Databases
Development tools

Examples
Azure App Service
AWS Elastic Beanstalk

Benefits
Faster development
Reduced management

3.3 SaaS (Software as a Service)

Provides software applications through the internet.

Examples
Gmail
Microsoft 365
Benefits
No installation
Accessible anywhere
